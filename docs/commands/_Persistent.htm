#Persistent

Hält ein Script dauerhaft aufrecht (bis der Benutzer das Script schließt oder ein ExitApp vorkommt).

#Persistent

Diese Direktive (egal wo) wird das Script weiterlaufen lassen, wenn es das Ende des automatischen Ausführungsbereichs (oberster Bereich des Scripts) erreicht hat und alle anderen Threads beendet wurden. Diese Direktive ist normalerweise nicht nötig, weil das Script automatisch persistent wird, wenn das Programm einige Möglichkeiten zum Starten von neuen Threads findet, wie z. B. Hotkey, Timer oder GUI.

In einigen Fällen könnte diese Direktive erforderlich sein (wenn keine laufenden Threads oder Hotkeys, Timers usw. vorkommen):

  • Scripts, die RegisterCallback und DllCall verwenden, um auf Ereignisse zu reagieren, weil solche Funktionen das Script nicht persistent machen.
  • Scripts, die COM-Objekte erstellen oder abrufen, und ComObjConnect verwenden, um auf die Ereignisse des Objekts zu reagieren.

Sobald diese Direktive in ein vorhandenes Script eingefügt wird, empfiehlt es sich, einige oder alle Vorkommen von Exit mit ExitApp zu ersetzen. Das Exit würde hierbei nur den aktuellen Thread beenden, nicht das ganze Script.
