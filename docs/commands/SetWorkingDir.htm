SetWorkingDir

Ändert das Arbeitsverzeichnis des Scripts. 

SetWorkingDir, Verzeichnisname

Parameter

Verzeichnisname

Name des neuen Arbeitsverzeichnisses, das ein Unterordner im aktuellen A_WorkingDir-Verzeichnis ist, solange kein absoluter Pfad angegeben ist.

ErrorLevel

This command is able to throw an exception on failure. For more information, see Runtime Errors.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Das Arbeitsverzeichnis des Scripts ist das Standardverzeichnis, das für den Zugriff auf Dateien und Ordner verwendet wird, falls kein absoluter Pfad angegeben ist. In the following example, the file My Filename.txt is assumed to be in %A_WorkingDir%: FileAppend, Eine Textzeile., Dateiname.txt.

The script's working directory defaults to A_ScriptDir, regardless of how the script was launched. By contrast, the value of A_InitialWorkingDir is determined by how the script was launched. For example, if it was run via shortcut -- such as on the Start Menu -- its initial working directory is determined by the "Start in" field within the shortcut's properties.

Diese Änderung wirkt sich sofort auf das gesamte Script aus. Alle unterbrochenen, pausierten und neu ausgeführten Threads sind davon betroffen, einschließlich Timer.

Siehe auch

A_WorkingDir, A_InitialWorkingDir, A_ScriptDir, DirSelect

Beispiel

SetWorkingDir %A_ScriptDir%
SetWorkingDir, D:\Eigener Ordner\Temp
