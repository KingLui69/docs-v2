OnExit

Bestimmt eine Funktion, die automatisch aufgerufen werden soll, wenn das Script beendet wird.

OnExit([CallbackFunk])

Parameter

CallbackFunk

Die Funktion, die aufgerufen werden soll, wenn das Script beendet wird - entweder als Name oder Verweis.

Gebe einen leeren String an, um die aktuelle Funktion zu entfernen. Fehlt dieser Parameter komplett, wird sich nichts ändern.

Rückgabewert

Gibt einen Verweis auf eine Funktion zurück, wenn eine Funktion zuvor gesetzt wurde, ansonsten ein leerer String.

CallbackFunk(ExitReason)

Das Script wird beendet, nachdem die Callback-Funktion aufgerufen wurde, sofern die Funktion ungleich True zurückgibt. Das Aufrufen von ExitApp innerhalb der Funktion wird das Script sofort beenden, ohne dass die Callback-Funktion nochmals aufgerufen wird.

Die Callback-Funktion akzeptiert optional einen einzelnen Parameter. Wenn der Exit- oder ExitApp-Befehl verwendet wurde, enthält dieser Parameter den Exitcode. Ansonsten enthält er einen der folgenden Strings, auf welche Weise das Script beendet wird:

Logoff Der Benutzer meldet sich ab.
Shutdown Das System wird heruntergefahren oder neu gestartet (z. B. mit dem Shutdown-Befehl).
Close Das Script hat eine WM_CLOSE- oder WM_QUIT-Nachricht empfangen, hatte einen kritischen Fehler, oder wurde auf andere Weise geschlossen. Zwar sind sie alle ungewöhnlich, allerdings kann es sein, dass WM_CLOSE aufgrund von WinClose erzeugt wird, wenn man diesen Befehl auf das Hauptfenster des Scripts anwendet. Um das zu verhindern, muss das Hauptfenster mit Send, !{F4} geschlossen werden.
Error Es ist ein Laufzeitfehler in einem Script entstanden, das nicht persistent ist. Ein Laufzeitfehler ist beispielsweise ein Run/RunWait, der das angegebene Programm oder Dokument nicht öffnen kann.
Menu Der Benutzer hat Exit im Hauptfenster-Menü oder voreingestellten Tray-Menü ausgewählt.
Reload Das Script wird via Reload-Befehl oder -Menüpunkt neu geladen.
Single Das Script wird aufgrund von #SingleInstance mit einer neuen Instanz von sich selbst ersetzt.

Bemerkungen

Die Callback-Funktion wird aufgerufen, sobald das Script in irgendeiner Form beendet wird (es sei denn, das Script wird mit einer gewaltsamen Methode beendet, wie z. B. "Task beenden"). Sie wird auch aufgerufen, wann immer #SingleInstance und Reload eine vorherige Instanz zum Beenden auffordert.

Mit OnMessage(0x11, "WM_QUERYENDSESSION") kann das Herunterfahren oder Abmelden des Betriebssystems erkannt und optional abgebrochen werden.

Der OnExit-Thread nimmt keine Rücksicht auf #MaxThreads (er wird bei Bedarf immer gestartet). Darüber hinaus kann dieser Thread, während er läuft, nicht von anderen Threads unterbrochen werden, einschließlich Hotkeys, benutzerdefinierte Menüpunkte und zeitgesteuerte Subroutinen. Allerdings wird er unterbrochen (und das Script beendet), wenn der Benutzer den Menüpunkt Exit im Tray-Menü oder Hauptmenü auswählt, oder das Script aufgrund von Reload oder #SingleInstance zum Beenden aufgefordert wird. Deshalb sollte die Callback-Funktion schnell zum Ende kommen, solange dem Benutzer nicht bewusst ist, was er tut.

Wenn der OnExit-Thread auf einen fehlerhaften Zustand wie z. B. Laufzeitfehler stößt, wird das Script beendet. Dadurch wird verhindert, dass das Script aufgrund einer fehlerhaften Callback-Funktion nicht mehr beendet werden kann.

Sobald die Callback-Funktion von einem Exitversuch aufgerufen wird, beginnt sie mit den Standardwerten in Einstellungen wie SendMode. Diese Standardeinstellungen können im automatischen Ausführungsbereich geändert werden.

Beispiel

#Persistent  ; Um das Ergebnis beim Drücken von "Nein" zu sehen, wird das Script aufrecht erhalten.
OnExit("ExitFunk")
return

ExitFunk(ExitReason)
{
    if ExitReason != "Logoff" and ExitReason != "Shutdown"
    {
        MsgBox, 4, , Möchten Sie das Script wirklich beenden?
        if A_MsgBoxResult = "No"
            return true  ; Verhindert, dass das Script beendet werden kann.
    }
    ; Keinen Rückgabewert: das Script wird automatisch beendet.
}
