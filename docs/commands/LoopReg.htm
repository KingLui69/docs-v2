Loop (Registrierung)

Ermittelt nacheinander den Inhalt des angegebenen Unterschlüssels.

Loop, Reg, RootKey\Key [, Mode]

Parameter

Reg

This parameter must be the word REG, and cannot be an expression or variable reference.

Hauptschlüssel

Entweder HKEY_LOCAL_MACHINE (oder HKLM), HKEY_USERS (oder HKU), HKEY_CURRENT_USER (oder HKCU), HKEY_CLASSES_ROOT (oder HKCR) oder HKEY_CURRENT_CONFIG (oder HKCC).

To access a remote registry, prepend the computer name and a slash, as in this example:
\\workstation01\HKEY_LOCAL_MACHINE

Schlüssel

Name des Schlüssels (z. B. Software\BeliebigeAnwendung). Ist dieser Parameter leer oder nicht vorhanden, wird der Inhalt vom Hauptschlüssel abgerufen.

Modus

One or more of the following characters (case-insensitive), optionally separated by spaces or tabs:

V: Retrieve values which match the pattern. This is the default if "D" is not specified.
K: Retrieve subkeys which match the pattern.
R: Recurse into subkeys so that all values and subkeys contained therein are retrieved.

Bemerkungen

Eine Registrierungs-Schleife ist nützlich, wenn mehrere Werte und Unterschlüssel nacheinander bearbeitet werden sollen. Die Werte und Unterschlüssel werden in umgekehrter Reihenfolge abgerufen (von unten nach oben), damit RegDelete ohne Unterbrechung innerhalb der Schleife verwendet werden kann.

Die folgenden Variablen können nur innerhalb einer Registrierungs-Schleife verwendet werden. Wenn eine innere Registrierungs-Schleife von einer äußeren Registrierungs-Schleife umschlossen ist, wird der Eintrag von der innersten Schleife Vorrang haben.

A_LoopRegName Name des aktuell ermittelten Eintrags, der entweder ein Wertname oder der Name eines Unterschlüssels sein kann. Der im Registrierungs-Editor angezeigte Wertname "(Standard)" wird abgerufen, wenn ihm ein Wert zugewiesen wurde - A_LoopRegName selbst wird aber leer sein.
A_LoopRegType Typ des aktuell ermittelten Eintrags, der eins der folgenden Wörter sein kann: KEY (bedeutet, dass der aktuell ermittelte Eintrag ein Unterschlüssel ist, aber kein Wert), REG_SZ, REG_EXPAND_SZ, REG_MULTI_SZ, REG_DWORD, REG_QWORD, REG_BINARY, REG_LINK, REG_RESOURCE_LIST, REG_FULL_RESOURCE_DESCRIPTOR, REG_RESOURCE_REQUIREMENTS_LIST, REG_DWORD_BIG_ENDIAN (wahrscheinlich selten in den meisten Betriebssystemen). Diese Variable ist leer, wenn der aktuell ermittelte Eintrag ein unbekannter Typ ist.
A_LoopRegKey Name des zurzeit verwendeten Hauptschlüssels (HKEY_LOCAL_MACHINE, HKEY_USERS, HKEY_CURRENT_USER, HKEY_CLASSES_ROOT oder HKEY_CURRENT_CONFIG). For remote registry access, this value will not include the computer name.
A_LoopRegSubKey Name des aktuellen Unterschlüssels. Ist das gleiche wie der Schlüssel-Parameter, solange der Rekursiv-Parameter nicht dazu verwendet wird, andere Unterschlüssel zu durchsuchen. In diesem Fall enthält die Variable den vollständigen Pfad des aktuell ermittelten Eintrags, ohne Hauptschlüssel. Zum Beispiel: Software\BeliebigeAnwendung\EigenerUnterschlüssel
A_LoopRegTimeModified Wann der aktuelle Unterschlüssel oder seine Werte zuletzt bearbeitet wurden. YYYYMMDDHH24MISS-Format. Diese Variable wird leer sein, wenn der aktuell ermittelte Eintrag kein Unterschlüssel ist (d. h. A_LoopRegType ist nicht das Wort KEY).

Innerhalb einer Registrierungs-Schleife können folgende Befehle auf einfachere Weise verwendet werden, um den aktuell ermittelten Eintrag zu bearbeiten:

RegRead, AusgabeVar Liest den aktuellen Eintrag. Wenn der aktuelle Eintrag ein Schlüssel ist, wird Errorlevel auf 1 gesetzt und AusgabeVar leer gemacht.
RegWrite [, Wert] Schreibt in den aktuellen Eintrag. Falls Wert nicht vorhanden ist, wird der Eintrag je nach Typ auf 0 gesetzt oder leer gemacht. Wenn der aktuelle Eintrag ein Schlüssel ist, wird ErrorLevel auf 1 gesetzt.
RegDelete Löscht den aktuellen Eintrag. Wenn der aktuelle Eintrag ein Schlüssel ist, wird dieser mit all seinen Unterschlüsseln und Werten gelöscht.

Beim Zugriff auf eine Remote-Registrierung (über den Hauptschlüssel-Parameter, wie oben beschrieben) gelten folgende Hinweise:

  • Auf dem Zielrechner muss der Dienst "Remote Registry" laufen.
  • Der Zugriff auf eine Remote-Registrierung kann fehlschlagen, wenn der Zielcomputer nicht zur Domäne des Benutzers gehört oder der Benutzername nicht genügend Berechtigungen hat (siehe unten für mögliche Übergangslösungen).
  • Abhängig von der Domäne, der Arbeitsgruppe und/oder den Berechtigungen kann es sein, dass sich der Benutzer mit einem freigegebenen Gerät erst verbinden muss (z. B. durch eine Laufwerkszuordnung), bevor der Remotezugriff auf die Registrierung erfolgt. Bei so einer Verbindung (mit Benutzernamen, Passwort und genug Berechtigungen zur Bearbeitung der Registrierung) könnte als Nebeneffekt der Remotezugriff auf die Registrierung aktiviert werden.
  • Wenn der Benutzer bereits mit einem anderen Benutzernamen auf dem Zielcomputer angemeldet ist (z. B. als Gast auf ein zugeordnetes Laufwerk), dann kann es sein, dass der Benutzer sich erst abmelden muss, damit die Remote-Registrierung sich neu verbinden kann und den Benutzer wieder mit seinem eigenen angemeldeten Benutzernamen authentifiziert.
  • Für Windows Server 2003 und Windows XP Professional heißt es in MSDN: "Wenn der [Server der Registrierung] Computer mit einer Arbeitsgruppe verbunden ist und die Sicherheitsrichtlinie Einfache Dateifreigabe aktiviert ist, dann schlägt die Funktion fehl. Beachten Sie, dass diese Sicherheitsrichtlinie standardmäßig aktiviert ist, wenn der Computer mit einer Arbeitsgruppe verbunden ist."
  • Für Windows XP Home Edition weißt MSDN darauf hin, dass "diese Funktion immer fehlschlägt". In der Home Edition fehlt für die Registrierung sowohl der Server als auch der Client, jedoch kann der Client aus einer CAB-Datei des Betriebssystems extrahiert werden.

Auf der Loop-Seite können Informationen zu Blöcke, Break, Continue und A_Index gefunden werden (weil diese in jeder Schleifenvariante vorkommen).

Beispiele

; Beispiel: Delete Internet Explorer's history of URLs typed by the user:
Loop, Reg, HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\TypedURLs
    RegDelete

 

; Beispiel: A working test script:
Loop, Reg, HKCU\Software\Microsoft\Windows, R KV  ; Recursively retrieve keys and values.
{
    if a_LoopRegType = "key"
        value := ""
    else
    {
        RegRead, value
        if ErrorLevel
            value := "*error*"
    }
    MsgBox, 4, , %a_LoopRegName% = %value% (%a_LoopRegType%)`n`nContinue?
    if A_MsgBoxResult = "No", break
}

 

; Beispiel: A working example to recursively search the entire
; registry for particular value(s).
RegSearch("Notepad")
return

RegSearch(Target)
{
    ContinueRegSearch := true
    Loop, Reg, HKEY_LOCAL_MACHINE, KVR
    {
        Gosub, CheckThisRegItem
        if !ContinueRegSearch  ; It told us to stop.
            return
    }
    Loop, Reg, HKEY_USERS, KVR
    {
        Gosub, CheckThisRegItem
        if !ContinueRegSearch  ; It told us to stop.
            return
    }
    Loop, Reg, HKEY_CURRENT_CONFIG, KVR
    {
        Gosub, CheckThisRegItem
        if !ContinueRegSearch  ; It told us to stop.
            return
    }
    ; Note: I believe HKEY_CURRENT_USER does not need to be searched if HKEY_USERS is
    ; being searched.  Similarly, HKEY_CLASSES_ROOT provides a combined view of keys from
    ; HKEY_LOCAL_MACHINE and HKEY_CURRENT_USER, so searching all three isn't necessary.
    return

CheckThisRegItem:
    if A_LoopRegType = "KEY"  ; Remove these two lines if you want to check key names too.
        return
    RegRead, RegValue
    if ErrorLevel
        return
    if InStr(RegValue, Target)
    {
        MsgBox, 4, , The following match was found:`n%A_LoopRegKey%\%A_LoopRegSubKey%\%A_LoopRegName%`nValue = %RegValue%`n`nContinue?
        if A_MsgBoxResult = "No"
            ContinueRegSearch := false  ; Tell our caller to stop searching.
    }
    return
}
