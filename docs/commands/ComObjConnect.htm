ComObjConnect()

Verbindet die Ereignisquellen eines COM-Objekts mit Funktionen, die einen vorgegeben Präfix enthalten.

ComObjConnect(ComObject [, Präfix])

Parameter

ComObject

Ein Objekt, das Ereignisse auslöst.

Präfix

Ein String, der als Präfix bei dem Ereignisnamen dienen soll, um feststellen zu können, welche Funktion bei einem Ereignis aufgerufen werden soll.

Ist dieser Parameter nicht vorhanden, wird das Objekt "getrennt"; das heißt, dass das Script nicht länger Benachrichtigungen von seinen Ereignissen erhält.

This parameter can be an object defined by the script. Beim Auslösen eines Ereignisses wird die entsprechende Methode aufgerufen. The first parameter, which is usually the hidden this parameter, refers to the script-defined object, not the COM object. To catch all events without defining a method for each one, define a __Call meta-function.

Verwendung

Um ComObjConnect effektiv zu nutzen, muss man erst Funktionen in das Script schreiben, die mit den gewünschten Ereignissen umgehen können. Solche Funktionen, auch "Ereignishandler" genannt, haben die folgende Struktur:

PräfixEreignisname([Params..., ComObject])
{
    ... Ereignis-behandelnder Code ...
    Return Rückgabewert
}

Für Präfix kann ein beliebiger Präfix angegeben werden, während der Ereignisname der Name des Ereignisses ist, das von der Funktion behandelt werden soll.

Params sind die Parameter des Ereignisses. Hat das Ereignis keine Parameter, sollte Params ganz weggelassen werden. ComObject ist optional und nur verwendbar, wenn die korrekte Anzahl von Params definiert wurde. Es enthält einen Verweis auf das originale Wrapper-Objekt, das an ComObjConnect übergeben wurde. "ComObject" sollte mit einem Namen ersetzt werden, der eher dem Kontext des Scripts entspricht.

Beachte, dass Ereignishandler Rückgabewerte haben können. To return a COM-specific type of value, use ComObject(VarType, Value).

Mit ComObjConnect(MeinObjekt, "Präfix") kann die Ereignisbehandlung aktiviert werden.

Mit ComObjConnect(MeinObjekt) kann das Objekt getrennt werden (die Ereignisbehandlung gestoppt werden).

Ist die Anzahl der Parameter unbekannt, kann eine variadische Funktion verwendet werden.

Beispiele

ie := ComObjCreate("InternetExplorer.Application")

; Connects events to corresponding script functions with the prefix "IE_".
ComObjConnect(ie, "IE_")

ie.Visible := true  ; This is known to work incorrectly on IE7.
ie.Navigate("http://ahkscript.org/")
#Persistent

IE_DocumentComplete(ieEventParam, url, ieFinalParam) {
    global ie
    if (ie != ieEventParam)
        s .= "First parameter is a new wrapper object.`n"
    if (ie == ieFinalParam)
        s .= "Final parameter is the original wrapper object.`n"
    if (ComObjValue(ieEventParam) == ComObjValue(ieFinalParam))
        s .= "Both wrapper objects refer to the same IDispatch instance.`n"
    MsgBox % s . "Finished loading " ie.Document.title " @ " url
    ie.Quit()
    ExitApp
}
