MenuSelect

Ruft einen Menüpunkt in der Menüleiste des angegebenen Fensters auf.

MenuSelect, WinTitle, WinText, Menu [, SubMenu1, SubMenu2, SubMenu3, SubMenu4, SubMenu5, SubMenu6, ExcludeTitle, ExcludeText]

Parameter

Fenstertitel

Ein Fenstertitel oder sonstige Kriterien, die das Zielfenster ermitteln sollen. Siehe Fenstertitel.

Fenstertext

In diesem Parameter kann ein Teilstring angegeben werden, der in einem Textelement des Zielfensters vorkommt (abrufbar mit Windows Spy). Versteckte Textelemente werden auch erkannt, sofern DetectHiddenText aktiv ist.

Menü

Name des Menüs auf höchster Ebene, z. B. Datei, Bearbeiten, Ansicht. Es ist auch möglich, die Position des gewünschten Menüpunkts anzugeben, indem 1& für das erste Menü verwendet wird, 2& für das zweite Menü, und so weiter.

Untermenü1

Name des Menüpunkts, das ausgewählt werden soll, oder dessen Position (siehe oben).

Untermenü2

Falls Untermenü1 selbst ein Menü enthält, ist dieser Parameter der Name des darin befindlichen Menüpunkts, oder dessen Position.

Untermenü3

Wie oben.

Untermenü4

Wie oben.

Untermenü5

Wie oben.

Untermenü6

Wie oben.

Titelausnahme

Ignoriert Fenster, deren Titel diesen Wert enthalten.

Textausnahme

Ignoriert Fenster, deren Textelemente diesen Wert enthalten.

ErrorLevel

This command is able to throw an exception on failure. For more information, see Runtime Errors.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Bemerkungen

Dieser Befehl funktioniert auch, wenn das Zielfenster nicht aktiv ist. Allerdings müssen sich einige Fenster möglicherweise in einem nicht minimierten Zustand befinden.

Dieser Befehl wird nicht funktionieren, wenn Programme nicht-standardmäßige Menüleisten verwenden. Beispiele sind Microsoft Outlook und Outlook Express, die versteckte Werkzeugleisten als Menüleisten verwenden. In solchen Fällen sind ControlSend oder PostMessage nützlich, die in der Regel mit einigen von diesen nicht-standardmäßigen Menüleisten interagieren können.

Die Menünamen unterscheiden nicht zwischen Groß- und Kleinschreibung (das heißt, dass Datei->Speichern das gleiche ist wie datei->speichern). Außerdem ist es nicht notwendig, ein Und-Zeichen (&) zu verwenden, um den unterstrichenen Buchstaben in einem Menüpunkt zu kennzeichnen (das heißt, dass &Datei das gleiche ist wie Datei).

Man kann in den Menü-Parametern auch Positionsnummern angeben. Diese Methode unterstützt Menüs, die keinen Text enthalten (weil sie beispielsweise Bilder mit Text statt reinen Text enthalten). Position 1& ist der erste Menüpunkt (z. B. das Datei-Menü), Position 2& ist der zweite Menüpunkt (z. B. das Bearbeiten-Menü), und so weiter. Menü-Trennlinien zählen als Menüpunkte zwecks Positionserkennung eines Menüpunkts.

Fenstertitel und -texte unterscheiden zwischen Groß- und Kleinschreibung. Solange DetectHiddenWindows deaktiviert ist, werden versteckte Fenster nicht wahrgenommen.

Siehe auch

ControlSend, PostMessage

Beispiel

; This will select File->Open in Notepad:
MenuSelect, Untitled - Notepad, , File, Open

; Same as above except it's done by position vs. name:
MenuSelect, Untitled - Notepad, , 1&, 2&
