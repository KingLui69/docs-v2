FileGetSize

Ermittelt die Größe einer Datei.

FileGetSize, AusgabeVar [, Dateiname, Einheit]

Parameter

AusgabeVar

Name der Variable, in der die ermittelte Größe gespeichert werden soll (auf die nächste ganze Zahl abgerundet).

Dateiname

Name der Datei, die sich im A_WorkingDir-Verzeichnis befindet, solange kein absoluter Pfad angegeben ist. Ist dieser Parameter nicht vorhanden, wird stattdessen die aktuelle Datei von der innersten umschließenden Loop (Dateien & Ordner) verwendet.

Einheit

Falls vorhanden, bewirkt dieser Parameter, dass das Ergebnis in einer bestimmten Einheit ausgegeben wird (standardmäßig in Bytes):
K = Kilobytes
M = Megabytes

ErrorLevel

This command is able to throw an exception on failure. For more information, see Runtime Errors.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Die interne Variable A_LastError enthält den Rückgabewert der Systemfunktion GetLastError().

Bemerkungen

Es werden Dateien in beliebiger Größe unterstützt, auch solche, die größer als 4 GB sind, und auch, wenn Einheit in Bytes ist.

Wenn Dateiname ein Verzeichnis ist, wird eine Größe abhängig vom Betriebssystem ausgegeben (in den meisten Fällen wahrscheinlich 0).

Im folgenden Beispiel wird gezeigt, wie die Größe des Ordners einschließlich seiner Dateien berechnet werden kann:

SetBatchLines, -1  ; Make the operation run at maximum speed.
FolderSize = 0
DirSelect, WhichFolder  ; Ask the user to pick a folder.
Loop, %WhichFolder%\*.*, , 1
    FolderSize += %A_LoopFileSize%
MsgBox Size of %WhichFolder% is %FolderSize% bytes.

Beispiel

FileGetSize, AusgabeVar, C:\Eigene Dokumente\test.doc  ; Ermittelt die Größe in Bytes.
