RegWrite

Schreibt einen Wert in die Registrierung.

RegWrite, ValueType, RootKey\SubKey [, ValueName, Value]

Parameter

Werttyp

Muss entweder REG_SZ, REG_EXPAND_SZ REG_MULTI_SZ, REG_DWORD oder REG_BINARY sein.

Hauptschlüssel

Entweder HKEY_LOCAL_MACHINE (oder HKLM), HKEY_USERS (oder HKU), HKEY_CURRENT_USER (oder HKCU), HKEY_CLASSES_ROOT (oder HKCR) oder HKEY_CURRENT_CONFIG (oder HKCC). To access a remote registry, prepend the computer name and a slash, as in this example: \\workstation01\HKEY_LOCAL_MACHINE

Unterschlüssel

Der Name des Unterschlüssels (z. B. Software\BeliebigeAnwendung). If SubKey does not exist, it is created (along with its ancestors, if necessary). Ist der Unterschlüssel leer, wird der Wert direkt in den Hauptschlüssel geschrieben (obwohl einige Betriebssysteme sich weigern könnten, in die oberste Ebene von HKEY_CURRENT_USER zu schreiben).

Wertname

Der Name des zu schreibenden Wertes. Ist dieser Parameter nicht vorhanden, wird der Standardwert von Unterschlüssel verwendet. Dieser wird in RegEdit als "(Standard)" angezeigt.

Wert

Der zu schreibende Wert. Ist dieser Parameter nicht vorhanden, wird je nach Werttyp standardmäßig ein leerer String oder eine 0 verwendet. Falls der Text zu lang ist, kann er in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um ihn besser lesbar und verwaltbar zu machen.

ErrorLevel

This command is able to throw an exception on failure. For more information, see Runtime Errors.

ErrorLevel wird bei Misserfolg auf 1 und bei Erfolg auf 0 gesetzt.

Die interne Variable A_LastError enthält den Rückgabewert der Systemfunktion GetLastError().

Bemerkungen

Wenn der Werttyp REG_DWORD enthält, sollte der Wert zwischen -2147483648 und 4294967295 (0xFFFFFFFF) liegen.

Der zu schreibende Wert eines REG_BINARY-Schlüssels muss ein String mit Hex-Zeichen sein. Zum Beispiel ist es mithilfe des Strings 01A9FF77 möglich, einen REG_BINARY-Wert von 01,a9,ff,77 zu schreiben.

Der zu schreibende Wert eines REG_MULTI_SZ-Schlüssels muss aus mehreren Komponenten bestehen, die jeweils mit einen Zeilenumbruchzeichen (`n) enden. Optional kann die letzte Komponente auch mit Zeilenumbruchzeichen enden. Leere Komponenten sind nicht erlaubt. Das heißt, dass zwei Zeilenumbruchzeichen nicht hintereinander erfolgen dürfen (`n`n), denn das würde bedeuten, dass ein Wert in die Registrierung geschrieben wird, der kürzer als erwartet ist.

Mit der Registrierungs-Schleife können mehrere Registrierungsschlüssel und -werte abgerufen und bearbeitet werden.

Bei den Bemerkungen zur Registrierungs-Schleife wird näher gebracht, wie man auf die Registrierung eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge von den 64-Bit-Bereichen der Registrierung in einem 32-Bit-Script oder umgekehrt gelesen und geschrieben werden.

Beispiele

RegWrite, REG_SZ, HKEY_LOCAL_MACHINE\SOFTWARE\TestKey, MyValueName, Test Value
RegWrite, REG_BINARY, HKEY_CURRENT_USER\Software\TEST_APP, TEST_NAME, 01A9FF77
RegWrite, REG_MULTI_SZ, HKEY_CURRENT_USER\Software\TEST_APP, TEST_NAME, Line1`nLine2
