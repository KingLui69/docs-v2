InputBox

Zeigt ein Fenster zur Eingabe eines Strings an.

InputBox, AusgabeVar [, Titel, Anzeigetext, Optionen, Standardwert]

Parameter

AusgabeVar

Name der Variable, in der die Benutzereingabe gespeichert werden soll.

Titel

Der Titel der InputBox. Ist dieser Parameter leer oder nicht vorhanden, wird standardmäßig der aktuelle Name des Scripts verwendet.

Anzeigetext

Text des Eingabefensters, der darauf hinweist, was eingegeben werden soll. Falls der Anzeigetext zu lang ist, kann dieser in mehreren kurzen Zeilen aufgeteilt werden (Fortsetzungsbereich), um die Lesbarkeit und Wartbarkeit zu verbessern.

Optionen

Ein String mit Optionen (ohne Unterscheidung zwischen Groß- und Kleinschreibung), die jeweils durch ein Leer- oder Tabulatorzeichen voneinander getrennt sind.

Xn Yn: Die Koordinaten X und Y des Dialogfensters. Zum Beispiel würde X0 Y0 das Fenster in die obere linke Ecke des Desktops verschieben. Sobald eine der Koordinaten nicht vorhanden ist, wird das Dialogfenster in dieser Abmessung zentriert. Jede Koordinate kann negativ sein, um das Dialogfenster teilweise oder vollständig über den Bildschirmrand hinaus zu positionieren.

Wn Hn: Die Breite und Höhe des Dialogfensters. Zum Beispiel wäre W375 H189 die Standardgröße.

T: Gibt das Zeitlimit in Sekunden an. Zum Beispiel wäre T10.0 zehn Sekunden. Der Parameter wird automatisch auf 2147483 (24.8 Tage) zurückgesetzt, falls dieser Wert überschritten wird. Sobald die Zeit abgelaufen ist, wird das Eingabefenster automatisch geschlossen und ErrorLevel auf 2 gesetzt. AusgabeVar enthält weiterhin die Benutzereingabe.

Password: Maskiert die Benutzereingabe. Das gewünschte Zeichen kann wie folgt angegeben werden: Password*

Standardwert

Ein String, der anfänglich im Eingabefeld des Dialogfensters angezeigt wird. Der Benutzer kann den String z. B. durch Drücken der Rücktaste ändern.

ErrorLevel

Dieser Befehl ist in der Lage, bei Misserfolg eine Ausnahme auszulösen. Für mehr Informationen, siehe Laufzeitfehler.

Siehe unten.

Bemerkungen

Mit diesem Dialogfenster kann der Benutzer einen Text eingeben und danach OK oder CANCEL (Abbrechen) drücken. Durch Ziehen der Rahmen kann dessen Größe geändert werden.

ErrorLevel wird auf 1 gesetzt, wenn der Benutzer den Button CANCEL drückt, auf 0, wenn der Benutzer OK drückt, oder auf 2, wenn das Dialogfenster die Zeit überschritten hat. In allen drei Fällen wird AusgabeVar auf den eingegebenen Wert gesetzt. Dadurch kann der Button CANCEL außer Abbrechen noch andere Funktionen ausführen, falls der Entwickler das wünscht.

Ein GUI-Fenster kann eine modale InputBox mittels Gui, +OwnDialogs anzeigen. Eine modale InputBox hindert den Benutzer daran, mit dem GUI-Fenster zu interagieren, bis die InputBox wieder geschlossen wird.

Beispiel

InputBox, Passwort, Passwort eingeben, (die Eingabe wird versteckt), password
InputBox, Benutzereingabe, Telefonnummer, Bitte geben sie eine Telefonnummer ein., w640 h480
if ErrorLevel
    MsgBox, Sie haben CANCEL gedrückt.
else
    MsgBox, Sie haben "%Benutzereingabe%" eingegeben.
