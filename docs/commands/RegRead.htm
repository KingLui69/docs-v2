RegRead

Liest einen Wert aus der Registrierung.

RegRead, OutputVar, RootKey\SubKey [, ValueName]

Parameter

AusgabeVar

Name der Variable, in der der ermittelte Wert gespeichert werden soll. Falls der Wert nicht abgerufen werden kann, wird die Variable leer gemacht und ErrorLevel auf 1 gesetzt.

Hauptschlüssel

Entweder HKEY_LOCAL_MACHINE (oder HKLM), HKEY_USERS (oder HKU), HKEY_CURRENT_USER (oder HKCU), HKEY_CLASSES_ROOT (oder HKCR) oder HKEY_CURRENT_CONFIG (oder HKCC). To access a remote registry, prepend the computer name and a slash, as in this example: \\workstation01\HKEY_LOCAL_MACHINE

Unterschlüssel

Der Name des Unterschlüssels (z. B. Software\BeliebigeAnwendung).

Wertname

Der Name des zu ermittelnden Wertes. Ist dieser Parameter nicht vorhanden, wird der Standardwert von Unterschlüssel abgerufen. Dieser wird in RegEdit als "(Standard)" angezeigt. Wenn kein Standardwert vorhanden ist (d. h., wenn in RegEdit "Wert nicht festgelegt" anzeigt wird), wird AusgabeVar leer gemacht und ErrorLevel auf 1 gesetzt.

ErrorLevel

This command is able to throw an exception on failure. For more information, see Runtime Errors.

ErrorLevel wird bei Misserfolg auf 1 (z. B. nicht vorhandener Schlüssel oder Wert) oder bei Erfolg auf 0 gesetzt.

Die interne Variable A_LastError enthält den Rückgabewert der Systemfunktion GetLastError().

Bemerkungen

Zurzeit werden nur folgende Wert-Typen unterstützt: REG_SZ, REG_EXPAND_SZ, REG_MULTI_SZ, REG_DWORD und REG_BINARY.

REG_DWORD-Werte werden immer als positive Dezimalzahlen ausgedrückt.

Das Ergebnis eines REG_BINARY-Schlüssels ist ein String mit Hex-Zeichen. Zum Beispiel wird der REG_BINARY-Wert 01,a9,ff,77 als String 01A9FF77 gelesen.

Das Ergebnis eines REG_MULTI_SZ-Schlüssels besteht aus mehreren Komponenten, die jeweils mit einem Zeilenumbruchzeichen (`n) enden. Wenn keine Komponenten vorhanden sind, wird AusgabeVar leer gemacht. See FileSelect for an example of how to extract the individual components from OutputVar.

Mit der Registrierungs-Schleife können mehrere Registrierungsschlüssel und -werte abgerufen und bearbeitet werden.

Bei den Bemerkungen zur Registrierungs-Schleife wird näher gebracht, wie man auf die Registrierung eines Remotecomputers zugreifen kann.

Mit SetRegView können Einträge von den 64-Bit-Bereichen der Registrierung in einem 32-Bit-Script oder umgekehrt gelesen und geschrieben werden.

Beispiel

RegRead InstallDir, HKEY_LOCAL_MACHINE\Software\AutoHotkey, InstallDir
if ErrorLevel
    MsgBox Unable to detect AutoHotkey installation.
else
    MsgBox AutoHotkey is installed in: %InstallDir%
