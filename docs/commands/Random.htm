Random

Erzeugt eine Pseudo-Zufallszahl.

Random, AusgabeVar [, Min, Max]
Random, , NewSeed

Parameter

AusgabeVar

Name der Variable, in der das Ergebnis gespeichert werden soll.

Min
The smallest number that can be generated, which can be negative or floating point. Ist dieser Parameter nicht vorhanden, wird die kleinste Zahl eine 0 sein. Der niedrigste zulässige Wert ist bei Integer -2147483648, jedoch bei Fließkommazahlen unbegrenzt.
Max

The largest number that can be generated, which can be negative or floating point. Ist dieser Parameter nicht vorhanden, wird die größte Zahl (und gleichzeitig auch der größte zulässige Wert) 2147483647 sein. Fließkommazahlen hingegen sind unbegrenzt.

NewSeed

This mode reseeds the random number generator with NewSeed. Das betrifft alle danach generierten Zufallszahlen. NewSeed should be an integer between 0 and 4294967295 (0xFFFFFFFF). Reseeding can improve the quality/security of generated random numbers, especially when NewSeed is a genuine random number rather than one of lesser quality such as a pseudo-random number. Generell muss dieses Verfahren nicht mehr als einmal durchgeführt werden.

Falls Reseeding nie vom Script durchgeführt wird, beginnt der Seed als niedrigwertige 32-Bits vom 64-Bit-Wert, der die Anzahl von 100-Nanosekunden-Intervallen seit 1. Januar 1601 enthält. Dieser Wert wandert alle ~7.2 Minuten von 0 bis 4294967295.

Bemerkungen

Dieser Befehl gibt eine Pseudo-Zufallszahl zurück, die eine echte Zufallszahl simuliert. In Wirklichkeit ist sie eine Zahl, die von einer komplizierten Formel abgeleitet wird, um das Bestimmen/Erraten der nächsten Zahl zu erschweren.

Alle Zahlen innerhalb des angegebenen Bereichs haben ungefähr die gleich hohe Wahrscheinlichkeit, erzeugt zu werden (siehe aber "Bekannte Einschränkungen" weiter unten).

If either Min or Max contains a decimal point, the end result will be a floating point number. Ansonsten wird das Ergebnis ein Integer sein.

Bekannte Einschränkungen bei Fließkommazahlen: 1) Nur etwa 4294967296 verschiedene Zahlen können in einem bestimmten Bereich erzeugt werden, alle anderen Zahlen im Bereich werden somit nie erzeugt; 2) Gelegentlich kann ein Ergebnis etwas größer sein als der angegebene Max-Wert (teilweise durch die interne Ungenauigkeit von Fließkommazahlen).

Beispiele

Random, rand, 1, 10
Random, rand, 0.0, 1.0

Kommentare vom originalen Quellcode

Diese Funktion verwendet den Mersenne-Twister-Zufallszahlengenerator, MT19937, geschrieben von Takuji Nishimura und Makoto Matsumoto, Shawn Cokus, Matthe Bellew und Isaku Wada.

Der Mersenne-Twister ist ein Algorithmus zur Erzeugung von Zufallzahlen. Bei dessen Entwicklung wurden die Fehler anderer Generatoren berücksichtigt. Die Periode, 219937-1, und die Gleichverteilung, 623 Dimensionen, sind weitaus größer. Der Generator ist zudem schnell; er verhindert Multiplikation und Division, und profitiert von Caches und Pipelines. Mehr Informationen können auf der Entwickler-Webseite gefunden werden: www.math.keio.ac.jp/~matumoto/emt.html
